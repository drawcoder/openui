# ADR-0002: Canvas仪表盘网格布局策略

## 状态

已采纳

## 背景

Canvas中的DashboardTab展示DashboardCard组件。我们需要确定：
1. 布局模式（自由定位 vs 网格对齐）
2. 卡片排列算法
3. 用户交互能力（拖拽、调整大小）
4. 溢出处理（滚动 vs 分页）

## 决策

我们采用**12列固定网格布局**，具体策略：
- **静态定位**：卡片严格按DSL定义顺序排列
- **无用户交互**：用户不能拖拽或调整卡片大小
- **垂直滚动**：Canvas无限纵向扩展，溢出时滚动
- **自动换行**：剩余列不足时卡片换到下一行

### 布局参数

- 列数：固定12列
- 行高：动态（内容自适应）
- 默认卡片宽度：6列 (`{w: 6}`)
- 尺寸单位：网格列 `{w: number}` (1-12)
- 卡片高度：根据内容自动适应

### 排列算法

卡片从左到右填充，空间不足时换行：
```
第1行: [CardA(6)] [CardB(4)] [空白(2)]
       → CardC(8)无法放入，换行到第2行
第2行: [CardC(8)] [空白(4)]
第3行: [CardD(6)] [空白(6)]
```

严格保持定义顺序——不做紧凑填充重排。

## 理据

### 备选方案

1. **自由拖拽定位**
   - 用户可自由放置卡片
   - 重叠和混乱布局的风险
   - 需要位置持久化存储
   - 前端实现更复杂

2. **紧凑填充算法**
   - 重排卡片以最优填充行
   - 破坏DSL定义顺序
   - 用户预期顺序布局时产生困惑

3. **分页**
   - 每页固定卡片数
   - 需要用户导航
   - 丧失全局概览可见性

4. **缩放/比例**
   - 缩小比例概览
   - 增加复杂性但收益有限

### 为何选择静态网格

- **布局可预测**：LLM和用户确切知道卡片位置
- **实现更简单**：无需拖拽/放置、无需持久化、无需重叠逻辑
- **移动端友好**：网格自然适配，滚动通用有效
- **仪表盘惯例**：12列网格是仪表盘工具的标准（Bootstrap、Ant Design Pro）

## 影响

### 正面
- DSL定义决定确定性的布局
- 无需用户偏好存储
- 整洁有序的仪表盘外观
- 较低的前端复杂度

### 负面
- 生成后用户无法自定义布局
- 大量卡片需要滚动
- 无紧凑打包优化

### 缓解措施
- Prompt可警告DashboardCard数量过多
- CanvasTabs提供"清空画布"按钮供用户重置
- 默认6列宽度高效支持每行2张卡片
