# OpenUI GenUI Java SDK

Java 21 SDK for assembling GenUI system prompts from:

- the built-in DSLEngine base contract resource
- one registered `GenUIContextExtension` selected by `contextId`
- one request-scoped `GenUIPromptRequest` overlay

The default resource is generated by `packages/react-ui-dsl`:

```bash
pnpm --dir packages/react-ui-dsl run generate:base-contract
```

That command writes `src/main/resources/openui/base-contract.json`. The primary
entrypoint, `GenerationSdk.create()`, loads this resource automatically. Tests
can override it with `GenerationSdk.builder().baseContract(...).build()`.

## Prompt assembly authority & golden tests

`PromptAssembler` is the authoritative GenUI prompt assembler (design D9) and is a
byte-for-byte port of `packages/lang-core/src/parser/prompt.ts`'s `generatePrompt`.
Builtin docs come from the contract's top-level `builtins` manifest (exported by the
frontend); all other sections are transcribed from `prompt.ts`.

Cross-language alignment is pinned by `PromptGoldenTest`: each
`src/test/resources/prompt-golden/<name>.json` is a merged `PromptSpec`, and the
matching `<name>.txt` is produced by the TypeScript oracle. Regenerate the golden
outputs (and re-export the base contract when the builtins registry changes) with:

```bash
pnpm --dir packages/react-ui-dsl run generate:base-contract
pnpm --dir packages/lang-core run generate:prompt-golden
```

## Running the tests

Run the Java SDK tests with Maven:

```bash
cd packages/genui-java-sdk
mvn test
mvn -Dtest=PromptGoldenTest test
```
